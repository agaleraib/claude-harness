## 1. Bake the "no verbosity caps in agent prompts" rule into the harness

**Source:** [An update on recent Claude Code quality reports — Anthropic Engineering, 2026-04-23](https://www.anthropic.com/engineering/april-23-postmortem)

### Why it's relevant

Anthropic's postmortem traces a measurable, **multi-week, ~3% drop in Claude Code's coding-eval scores** to a single system-prompt addition that capped responses at "≤25 words between tool calls, ≤100 words for final responses." It went out on April 16 alongside Opus 4.7 and was rolled back on April 20. Two consequences for this harness:

1. **A previously implicit rule is now empirically grounded.** This harness already leans on the model's own writing style ("doesn't tell the model HOW to think — tells the model WHAT to check against", `README.md:10`) and has no word caps in any agent or skill (verified — see Verify line). The postmortem turns that decision from taste into a measurement: any future agent that adds a cap is paying a known 3% quality tax.
2. **The model-pin / effort-routing spec gets external validation.** `docs/specs/2026-04-19-harness-model-pin-and-effort-routing.md` already pins `effort_default: xhigh` and §1d carries an effort-promotion rung. The postmortem confirms (a) Anthropic restored "high" as the safe default on April 20, and (b) `effort=high` is the production-grade setting going forward. No new spec change is needed — but the spec should cite the postmortem as the public evidence so a future reader who skims the spec can see the pin isn't speculative.

### Concrete changes

Two small edits, no new files:

- **`.claude/agents/orchestrator.md`** — add a one-liner under the existing "Brief subagents thoroughly" guidance (around line 203) that names the anti-pattern explicitly:

  ```diff
   2. **Brief subagents thoroughly.** A subagent starts cold — it hasn't seen the conversation. Include file contents, patterns to follow, and the verification step. Bad briefs waste more than they save.
  +
  +   **Anti-pattern: word/length caps in subagent prompts.** Do not write "respond in ≤25 words" or "keep final response under 100 words" in a subagent brief. Anthropic's 2026-04-23 postmortem traced a 3% Claude Code coding-eval regression to exactly this style of cap. If a brief is too long, trim *content* (drop irrelevant context), not *response budget*.
  ```

- **`docs/specs/2026-04-19-harness-model-pin-and-effort-routing.md`** — add one citation line in the Phase 2 acceptance-criteria block, right under the "Default effort selection rule" bullet:

  ```diff
  - [ ] Default effort selection rule (added to routing guidelines): `low` for read-only / scanning / typo-fix tasks; `medium` for standard implementation; `high` for architecture / complex algorithms; `xhigh` for code-reviewer loops, multi-file migrations, and any task flagged `stakes.level: high` in `.harness-profile`.
  + - [ ] **Empirical anchor (added 2026-04-25):** Anthropic's [April 23 postmortem](https://www.anthropic.com/engineering/april-23-postmortem) confirms `effort=high` as the production-grade default on Claude Code itself (medium was rolled back April 7 after customer regressions). The spec's `effort_default: xhigh` is one tier above their consumer default and is the right setting for a harness whose primary loop is code-reviewer + generator.
  ```

That's it — two text edits, both inside files that already exist on this branch.

### Easy example

A future contributor writes a new agent and copies a pattern from a public LLM blog that says *"keep responses concise — under 50 words."* Without the rule, that line lands and the harness silently degrades. With the rule, the rule appears two lines below the brief-writing guidance and the contributor sees it before they ship.

### Expected payoff

Locks in a *known-cost* anti-pattern (3% coding regression — Anthropic-measured, not folklore) at the only point where it could realistically be re-introduced (subagent prompts). Spec citation makes the model-pin work defensible to a future reader who didn't follow the April 19 review thread.
