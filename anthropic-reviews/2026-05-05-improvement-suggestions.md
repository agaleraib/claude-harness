## 1. Add a prefix-stability guardrail to `AGENTS.md` "What to avoid"

**Source:** [Lessons from building Claude Code: Prompt caching is everything — Claude blog, 2026-04-30](https://claude.com/blog/lessons-from-building-claude-code-prompt-caching-is-everything)

### Why it's relevant

The post's central rule — *"Any change anywhere in the prefix invalidates everything after it. Design your entire system around this constraint."* — is a primary-source claim from the team that built Claude Code itself. Most of the post's advice is for authors who construct raw API requests (defer-loading tool stubs, choosing breakpoint placement, fork-side cache hygiene). One rule transfers cleanly to harness authors who only edit *files that Claude Code injects into its prefix*: keep those files free of dates, build IDs, environment-specific paths, and other dynamic content. Verbatim from the post: *"common breaking points include putting an in-depth timestamp in the static system prompt, shuffling tool order definitions non-deterministically, and updating parameters of tools."*

A spot-check of this repo confirms `CLAUDE.md` and `AGENTS.md` are already date-free (`grep -n "currentDate\|today\|Today" CLAUDE.md AGENTS.md` returns nothing — the `# currentDate` block visible at session-start is injected by Claude Code itself as a `system-reminder`, not authored in-file, which is exactly the recommended pattern). So this isn't a fix-an-existing-problem § — it's a *guardrail* §. Future skill authors (or future-me on a sleepy Tuesday) need a written rule pointing at the failure mode, because the failure is invisible: a date stamp in `SKILL.md` doesn't break anything functionally, it just silently degrades cache-hit rate across every session that loads that skill.

### Concrete change

Append one bullet to the `## What to avoid` block in `AGENTS.md` (currently `AGENTS.md:27–33`):

```diff
 ## What to avoid

 - Don't `git add -A` or `git add .` — it sweeps in secrets and unrelated work.
 - Don't merge feature branches without `--no-ff` — it loses history.
 - Don't write durable state outside the protocol artifacts above (no `~/.tmp_state`, no untracked sidecars).
 - Don't invent state when inputs are missing — stop and surface.
 - Don't bypass the receipt schema. Adapter divergence on receipt shape breaks cross-tool replay.
+- Don't bake dynamic content (timestamps, build IDs, absolute paths, per-session counters) into static prompt files (`CLAUDE.md`, `AGENTS.md`, any `SKILL.md`). Per [Anthropic's prompt-caching guidance](https://claude.com/blog/lessons-from-building-claude-code-prompt-caching-is-everything), prefix-instability invalidates the conversation-wide cache. Use runtime injection (Claude Code's `system-reminder`, hook output, or a fresh-read of `.harness-state/`) for anything that changes session-to-session.
```

That's the whole change — one bullet, one citation. No code edits, no skill rewrites.

### Why this is a guardrail and not a fix

Audited the obvious candidates and they're already clean:

- `grep -n "currentDate\|today\|Today" CLAUDE.md AGENTS.md` → no matches.
- `grep -l "$(date +%Y)" CLAUDE.md AGENTS.md` → no matches in the prompt files; the year-pattern hits in `skills/*/SKILL.md` are inside test-fixture content (`planning-loop/lib/test-fixtures/`) or example output (`commit/SKILL.md`'s example commit message), not body text that becomes part of a stable prefix.
- The dynamic `# currentDate` and `# claudeMd` blocks visible at session-start come from Claude Code's own context-injection layer, which already follows the post's rule — they're delivered as `system-reminder` content, not stitched into the static system prompt.

So today this repo is fine. The bullet exists so the next skill author (writing `skills/<new-skill>/SKILL.md` with a "as of 2026-05" reference, or hard-coding `~/Users/me/...` instead of a portable token) has a one-line rule to point at in review.

### What this run does **not** propose

The post's other lessons (defer_loading for tool stubs, breakpoint placement, fork-cache parity) all address authors who hand-build `messages` arrays for the Anthropic API. claude-harness is a *Claude Code* harness — it doesn't construct API requests itself, so those lessons have no surface here. Skipping them is correct, not an omission.

### Expected payoff

Cheap durable hygiene. Costs one bullet today; pays off the first time a contributor (or future-you) reaches for `date +%Y-%m-%d` while editing a SKILL.md and remembers the rule. No measurable cache-hit win expected today since the repo is already compliant.

**Verify before applying:**
1. `grep -n "prompt-caching\|prefix" AGENTS.md` should return nothing — if a guardrail is already in place, this § is stale.
2. `grep -rn "currentDate\|todays.date\|build.id" CLAUDE.md AGENTS.md skills/*/SKILL.md` should still return only test-fixture / example matches; if a real prompt file has acquired a hard-coded date since this § was written, the suggestion is now a *fix*, not a guardrail, and §1 should be re-scoped to remove that content.

**Recommended verdict:** apply — one-bullet guardrail with a primary-source citation; no design risk, no code changes, durable as documentation.
