## 1. Adopt the BioMysteryBench "human-solvable / human-difficult" tier split when the deferred agent-evals fixtures land

**Source:** [Evaluating Claude's bioinformatics research capabilities with BioMysteryBench — Anthropic Research, 2026-04-29](https://www.anthropic.com/research/Evaluating-Claude-For-Bioinformatics-With-BioMysteryBench)

### Why it's relevant
The 2026-04-19 §5 (deferred) proposed `criteria/agent-evals/` fixtures for `code-reviewer` and `ui-evaluator`. The BioMysteryBench post reveals a useful design choice that *should* inform that fixture set whenever it's built: Anthropic split their 99-question benchmark into two graded tiers — *human-solvable* (the floor: a panel of five domain experts can produce the answer) and *human-difficult* (the ceiling: no expert on the panel solved it). They then report two scores per model — one per tier. The signal that comes out of that stratification is much sharper than a flat 1–10 score: a model that solves 82% on the floor but 0% on the ceiling is "useful, not superhuman"; one that scores 30% on the ceiling is doing something experts can't.

The current `criteria/*.md` rubrics in this repo (`code-architecture.md`, `data-integrity.md`, `frontend-ui-design.md`, `performance-accessibility.md`, `ux-user-flows.md`) all use a flat 1–10 scale per dimension. That works for grading a single artifact, but it doesn't tell you whether a reviewer agent is *catching the obvious bugs* (floor) or *catching the bugs a senior reviewer would miss* (ceiling). Two tiers per fixture would.

This is **not** an ask to retrofit the existing per-artifact rubrics — those are downstream-of-feature graders and the flat scale is fine. It's an ask to remember the floor/ceiling pattern when the deferred §5 fixture set is eventually scaffolded, where the value of the pattern is highest.

### Concrete change
**None this run.** Pure annotation against the existing 2026-04-19 §5 entry, parallel to how 2026-04-28 §3 worked. When §5 promotes from `DEFERRED` to `SPEC'd` or `APPLIED`, the spec author should consider stratifying fixtures into `floor/` and `ceiling/` subdirectories:

```
criteria/agent-evals/
  code-reviewer/
    floor/                       # bugs a 5-min code review catches; reviewer MUST flag
      01-missing-null-check.input/
      01-missing-null-check.expected
    ceiling/                     # bugs only a senior would catch; reviewer beats human-baseline if it flags
      01-subtle-race-condition.input/
      01-subtle-race-condition.expected
  ui-evaluator/
    floor/   # broken login flow, missing error states
    ceiling/ # accessibility regressions only a screen-reader user would feel
```

Score reporting would then be `floor: N/M, ceiling: N/M` per agent — matching BioMysteryBench's reporting format and making "is the agent at the human floor or beyond the human ceiling?" answerable.

The 2026-04-19 §5 is currently `DEFERRED` with no design notes attached. If it stays deferred for another two quarters, the BioMysteryBench tiering insight will be lost from the routine's reading. Capturing it here means a future reader who triggers §5 has one extra design constraint already filed.
